# 数据库表结构分析文档

本文档旨在清晰地展示用户提供的三个数据库表（`papers`、`cleaned` 和 `journal_metrics`）的结构、字段含义及示例数据。

## 1. 表结构概览

数据库中包含三张表，用于存储和管理学术论文及其相关的期刊指标数据。

| 表名 | 描述 |
| :--- | :--- |
| `papers` | 原始论文数据表，用于存储从外部源采集的未经清洗的论文信息。 |
| `cleaned` | 清洗后的论文数据表，结构与 `papers` 相同，用于存储经过数据清洗、规范化和预处理后的论文信息。 |
| `journal_metrics` | 期刊指标表，用于存储期刊在不同年份的各项学术计量指标。 |

---

## 2. 论文数据表 (`papers` 和 `cleaned`)

`papers` 和 `cleaned` 两张表的结构完全相同，区别在于存储的数据状态（原始 vs. 清洗）。

**SQL 建表语句 (以 `papers` 为例):**

```sql
CREATE TABLE papers (
  id INT AUTO_INCREMENT PRIMARY KEY,
  doi VARCHAR(255),
  journal VARCHAR(255),
  keywords TEXT,
  publish_date YEAR,
  target VARCHAR(255),
  citations TEXT,
  title TEXT,
  abstract TEXT,
  category VARCHAR(100),
  citing TEXT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 2.1 字段说明

| 字段名 | 数据类型 | 是否主键 | 含义说明 | 示例数据 |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `INT` | 是 | 论文记录的唯一自增标识符。 | `1` |
| `doi` | `VARCHAR(255)` | 否 | 论文的数字对象唯一标识符（Digital Object Identifier）。 | `10.16603/j.issn1002-1027.2015.06.017` |
| `journal` | `VARCHAR(255)` | 否 | 论文发表的期刊名称。 | `大学图书馆学报` |
| `keywords` | `TEXT` | 否 | 论文的关键词，通常以特定分隔符（如 `;;`）分隔。 | `Web2.0;;图书馆2.0` |
| `publish_date` | `YEAR` | 否 | 论文的发表年份。 | `2006` |
| `target` | `VARCHAR(255)` | 否 | 目标字段，具体用途待定（在示例数据中为空）。 | (空) |
| `citations` | `TEXT` | 否 | 论文的参考文献列表。 | `1　德国之声深度报道:什么是Web2.0?...` |
| `title` | `TEXT` | 否 | 论文的标题。 | `图书馆2.0:构建新的图书馆服务` |
| `abstract` | `TEXT` | 否 | 论文的摘要内容。 | (未在示例中明确展示) |
| `category` | `VARCHAR(100)` | 否 | 论文的分类或所属领域。 | `Chinese` |
| `citing` | `TEXT` | 否 | 引用该论文的文献列表或相关信息。 | (未在示例中明确展示) |

## 3. 期刊指标表 (`journal_metrics`)

该表用于存储特定期刊在不同年份的各项学术计量指标，主要用于对期刊进行量化分析。

**SQL 建表语句:**

```sql
CREATE TABLE IF NOT EXISTS journal_metrics (
  journal            VARCHAR(512) NOT NULL,
  year               INT NOT NULL,
  disruption         DOUBLE NULL,
  interdisciplinary  DOUBLE NULL,
  novelty            DOUBLE NULL,
  topic              DOUBLE NULL,
  theme_concentration DOUBLE NULL,
  hot_response       DOUBLE NULL,
  top_keywords_2021       LONGTEXT NULL,
  top_keywords_2022       LONGTEXT NULL,
  top_keywords_2023      LONGTEXT NULL,
  top_keywords_2024      LONGTEXT NULL,
  top_keywords_2025      LONGTEXT NULL,
  PRIMARY KEY (journal, year),
  paper_count         INT NOT NULL,
  category VARCHAR(100)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.1 字段说明

| 字段名 | 数据类型 | 是否主键 | 含义说明 |
| :--- | :--- | :--- | :--- |
| `journal` | `VARCHAR(512)` | 联合主键 | 期刊名称。 |
| `year` | `INT` | 联合主键 | 指标统计的年份。 |
| `disruption` | `DOUBLE` | 否 | 衡量期刊的颠覆性或创新性指标。 |
| `interdisciplinary` | `DOUBLE` | 否 | 衡量期刊的跨学科性指标。 |
| `novelty` | `DOUBLE` | 否 | 衡量期刊的新颖性指标。 |
| `topic` | `DOUBLE` | 否 | 衡量期刊的主题相关性指标。 |
| `theme_concentration` | `DOUBLE` | 否 | 衡量期刊的主题集中度指标。 |
| `hot_response` | `DOUBLE` | 否 | 衡量期刊的热点响应速度或程度指标。 |
| `top_keywords_2021` | `LONGTEXT` | 否 | 2021年期刊的顶级关键词列表。 |
| `top_keywords_2022` | `LONGTEXT` | 否 | 2022年期刊的顶级关键词列表。 |
| `top_keywords_2023` | `LONGTEXT` | 否 | 2023年期刊的顶级关键词列表。 |
| `top_keywords_2024` | `LONGTEXT` | 否 | 2024年期刊的顶级关键词列表。 |
| `top_keywords_2025` | `LONGTEXT` | 否 | 2025年期刊的顶级关键词列表。 |
| `paper_count` | `INT` | 否 | 该期刊在该年份发表的论文总数。 |
| `category` | `VARCHAR(100)` | 否 | 期刊的分类或所属领域。 |
